# News RAG

Учебный проект по анализу новостей. Цель первой версии: собрать новости из одного источника, сохранить их в базу данных, выполнить базовый анализ текстов и показать результаты в простом интерфейсе.

## MVP

Первая версия проекта включает:

- парсер новостей РБК;
- SQLite для локальной разработки и хранения статей;
- ETL-пайплайн `parse -> clean -> save`;
- заменяемый аналитический блок для тем, паттернов и динамики;
- интерфейс на Streamlit для просмотра новостей и результатов анализа.

ML-блок намеренно оставлен гибким. На старте можно реализовать простой анализ тем или кластеризацию, а позже заменить это на BERTopic, эмбеддинги, RAG или другой подход без полной переделки проекта.

PostgreSQL не убираем из архитектуры насовсем: при необходимости проект можно будет перевести на него позже простой заменой `DATABASE_URL` и миграциями.

## Архитектура

```text
news_rag/
  app/
    db/           # подключение к БД, базовые модели, сессии
    models/       # ORM-модели и схемы предметной области
    parsers/      # сбор новостей из внешних источников
    services/     # загрузка данных, ETL, orchestration
    ml/           # аналитика, темы, паттерны, эмбеддинги
    ui/           # Streamlit-интерфейс
    config.py     # настройки проекта
  scripts/        # служебные скрипты запуска
  tests/          # тесты
  alembic/        # миграции БД
  .env.example
  .gitignore
  requirements.txt
  README.md
```

## Первая схема данных

Основные сущности первой версии:

- `sources`:
  - `id`
  - `name`
  - `base_url`
- `articles`:
  - `id`
  - `source_id`
  - `url`
  - `title`
  - `overview`
  - `text`
  - `published_at`
  - `parsed_at`
  - `created_at`
- `topics`:
  - `id`
  - `name`
  - `keywords`
  - `created_at`
- `article_topics`:
  - `id`
  - `article_id`
  - `topic_id`
  - `score`
  - `created_at`

На старте можно реализовать только `sources` и `articles`, а таблицы аналитики добавить вторым этапом.

## Этапы разработки

1. Поднять каркас проекта и окружение.
2. Подключить локальную БД и описать модели данных.
3. Написать парсер РБК и сохранить статьи в БД.
4. Добавить базовый аналитический модуль.
5. Собрать интерфейс для просмотра новостей и аналитики.
6. Подготовить README, демо и итоговую презентацию.

## План на ближайшую неделю

1. Настроить зависимости и конфигурацию проекта.
2. Подключить БД и создать первую модель `Article`.
3. Сделать минимальный парсер РБК.
4. Реализовать сохранение статей в БД с дедупликацией по URL.
5. Подготовить простой скрипт запуска загрузки данных.

## Запуск

### 1. Создать виртуальное окружение

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. Создать `.env`

Скопировать `.env.example` в `.env` и при необходимости поправить параметры подключения.

По умолчанию проект использует локальный файл `news_rag.db`.

### 3. Инициализировать БД

Пока доступны два пути:

- быстрый старт через SQLAlchemy:

```powershell
python -m scripts.init_db
```

- дальнейший путь через Alembic:

```powershell
alembic revision --autogenerate -m "init schema"
alembic upgrade head
```

### 5. Запустить ingestion-заготовку

```powershell
python -m scripts.run_ingestion --query "РБК" --date-from 2024-04-22 --date-to 2024-04-23 --max-pages 1
```

### Переход на PostgreSQL позже

Если позже захочется перейти на PostgreSQL, можно:

1. установить Docker Desktop или локальный PostgreSQL;
2. поменять `DATABASE_URL` в `.env`;
3. использовать Alembic для миграций вместо `create_all`.
